第一層級：特徵與資料級別 (影響最大)
這是決定 MLP 能不能看到「正確答案」的最關鍵因素，比改神經網路本身還重要。

1. NUM_POIS (特徵點數量)
程式碼位置： main() 函式中的 NUM_POIS = 100

調整方式： 嘗試改為 50, 200, 300 甚至是 10。

影響結果：

調大 (例如 300)： 模型能接收到更多物理資訊。但如果 SNR 高的點其實只有 50 個，你硬塞 300 個點，等於主動餵給模型大量的雜訊，會導致模型難以收斂，甚至過擬合。

調小 (例如 10)： 輸入維度極低，訓練極快，完全去除了雜訊。但如果這 10 個點不足以拼湊出 S-Box 的完整運算特徵，模型就會「欠擬合 (Underfitting)」，永遠無法達到 Rank 0。

策略： 觀察生成的 SNR 圖表。如果高於某個基準線的突起點只有大約 50 個，那就設 50。

2. profiling_limit (訓練資料量)
程式碼位置： load_data_2d 的呼叫參數 profiling_limit=50000

調整方式： ASCADv2 資料集最大有幾十萬筆，如果你的 RAM 夠，可以往上拉到 100000 甚至 200000。

影響結果： 資料量是解決側信道攻擊的萬靈丹。 資料越多，模型越能看穿硬體的隨機雜訊，找到真正的電力洩漏特徵。這是提升準確率最暴力也最有效的方法。

第二層級：模型架構級別 (決定大腦容量)
如果特徵都找對了，但模型還是學不起來，或者學到一半就退步，就需要調整這裡。

3. 隱藏層的神經元數量 (Dense units)
程式碼位置： build_mlp_model 中的 layers.Dense(512) 與 layers.Dense(256)

調整方式：

擴建： 試著改成 1024 -> 512。

瘦身： 試著改成 256 -> 128 或 128 -> 128。

影響結果：

神經元越多，模型「死背」資料的能力越強。如果特徵很複雜，擴建可以幫助模型抓到非線性關係。但如果模型太寬，它會連雜訊一起背下來，導致驗證集準確率 (val_accuracy) 卡住。

對於只有 100 個 POIs 的輸入，512 -> 256 其實有點偏大，你可以嘗試往下瘦身，這能強迫模型去蕪存菁，找出真正的密鑰規律。

4. Dropout (隨機失活率)
程式碼位置： layers.Dropout(0.3) 與 0.2

調整方式： 調高至 0.4、0.5，或降低至 0.1、0.0。

影響結果：

這是防止過擬合的煞車皮。如果在訓練 Log 中，你看到 loss 一直下降，但 val_loss 卻在 Epoch 10 以後開始反向升高，這代表嚴重過擬合。此時請調高 Dropout。

反之，如果 loss 降得很慢，模型好像笨笨的學不起來，可能是煞車踩太重了，請調低 Dropout。

第三層級：訓練優化級別 (決定學習節奏)
5. learning_rate (學習率)
程式碼位置： tf.keras.optimizers.Adam(learning_rate=5e-4)

調整方式： 調大至 1e-3 (0.001)，或調小至 1e-4 (0.0001)。

影響結果：

調大 (1e-3)： 步伐邁得大，模型收斂極快，前幾個 Epoch Loss 就會掉很多。但缺點是可能在谷底附近反覆橫跳，找不到最佳解。

調小 (1e-4)： 步伐細膩，適合讓模型慢慢微調出完美的權重，但需要更多的 Epoch 才能達到終點。

6. batch_size (批次大小)
程式碼位置： model.fit() 中的 batch_size=256

調整方式： 調大至 512、1024，或調小至 64、128。

影響結果：

因為 MLP 參數極少，所以將 Batch Size 開得非常大（如 1024）。

大 Batch Size： 梯度更新非常穩定，計算速度極快。缺點是有時候會卡在「局部最佳解」出不來。

小 Batch Size： 每次更新會帶有一點雜訊，這種震盪有時能幫助模型跳出死胡同，但訓練時間會變長。

請按照以下順序進行實驗記錄：

先測基準線 (Baseline)： 直接跑一次現在的程式碼 (NUM_POIS=100, Dense(512,256))，記錄下 Byte 0 的 Final Rank。

尋找最佳視野 (NUM_POIS)： 保持模型不動，只改變 NUM_POIS (試試 50, 100, 200)，找出哪一個維度的 Rank 最低。

調整大腦容量 (Dense)： 用最佳的 NUM_POIS，去調整神經元數量 (試試 256->128 或 1024->512)，看模型是否能進一步突破。

最後微調 (LR & Dropout)： 如果還差一點點 (比如 Rank 卡在 2 或 3)，才去微調學習率或 Dropout 來擠出最後的效能。
